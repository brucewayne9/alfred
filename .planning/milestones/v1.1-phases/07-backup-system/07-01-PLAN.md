  <name>Task 1: Create Drive folder manager with create/upload/list/prune operations</name>
  <files>scripts/backup/drive_manager.py, scripts/backup/__init__.py</files>
  <action>
Create `scripts/backup/__init__.py` (empty package marker).

Create `scripts/backup/drive_manager.py` that wraps the existing `integrations/google_drive/client.py` functions for backup-specific folder management. The module must:

1. **`ensure_root_folder()`** — Find or create an "Alfred Backups" folder in Drive root. Cache the folder ID after first lookup to avoid repeated API calls. Use `list_files(query="Alfred Backups", file_type="folder")` to find existing, `create_folder("Alfred Backups")` if not found.

2. **`ensure_server_folder(server_name: str)`** — Find or create a subfolder for the given server under "Alfred Backups". Pattern: `Alfred Backups/{server_name}/`. Use same find-or-create pattern.

3. **`upload_backup(local_path: str, server_name: str, backup_type: str)`** — Upload a tar.gz file to `Alfred Backups/{server_name}/{backup_type}/YYYY-MM-DD/`. Creates the date folder, uploads the file. Returns the Drive file metadata dict. The `backup_type` is either "daily" or "weekly".

4. **`list_date_folders(server_name: str, backup_type: str)`** — List all date-stamped folders under a server's backup_type folder. Returns list of dicts with `id`, `name`, `modified` fields. Used by retention cleanup.

5. **`delete_folder_recursive(folder_id: str)`** — Delete a folder and its contents by trashing the folder (Drive cascades trash to children). Use `delete_file(folder_id)` from the existing client.

6. **`prune_old_backups(retention_days: int = 30)`** — For each server, for each backup_type (daily/weekly), list date folders, parse folder names as dates, delete folders older than `retention_days`. Log what was pruned. Return a summary dict of `{server: {deleted: [...], kept: [...]}}`.

Import from `integrations.google_drive.client` — do NOT duplicate Drive API code. Add `sys.path.insert(0, ...)` at top to resolve imports when run from scripts/ directory (pattern: `sys.path.insert(0, os.path.join(os.path.dirname(__file__), '..', '..'))`).

All functions should log operations via `logging.getLogger(__name__)`.
  </action>

  </verify>

2. **`run_cmd(alias: str | None, cmd: str, timeout: int = 60) -> str`** — Execute command locally or via SSH (same pattern as audit.py `run_cmd`). Return stdout. Raise on timeout/error with descriptive message including server alias and command.

3. **`collect_remote_files(alias: str | None, remote_paths: list[str], local_dest: str) -> list[str]`** — For each remote path, use `scp` (or local `cp`) to copy files to `local_dest/`. Skip files that don't exist (log warning). Return list of successfully copied file paths. For remote: `scp -o ConnectTimeout=10 {alias}:{path} {local_dest}/`. For local: `cp {path} {local_dest}/`.

4. **`pack_tarball(source_dir: str, output_path: str) -> str`** — Create a tar.gz from `source_dir`, writing to `output_path`. Return the output path. Use `tarfile` stdlib module.

5. **`DAILY_TARGETS`** — Dict mapping server name to list of file paths/globs to collect daily. Based on inventory findings:

6. **`WEEKLY_EXTRAS`** — Dict mapping server name to additional weekly-only collection commands:
