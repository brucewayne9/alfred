  <name>Task 1: Create audit script and collect server inventory</name>
  <files>data/infrastructure/audit.py, data/infrastructure/inventory.json</files>
  <action>
Create `data/infrastructure/` directory if it does not exist.

Write `data/infrastructure/audit.py` — a Python script that SSHes into each server using the aliases from `~/.ssh/config` and collects:

**Per-server data to collect (via SSH remote commands):**
1. **System info:** hostname, OS version (`cat /etc/os-release`), kernel (`uname -r`), uptime
2. **Disk usage:** `df -h` output for mounted filesystems
3. **Memory:** `free -h` output
4. **Docker containers:** `docker ps --format '{{.Names}}\t{{.Image}}\t{{.Status}}\t{{.Ports}}'` (handle case where Docker is not installed)
5. **Docker compose projects:** `docker compose ls --format json` if available
6. **Systemd services:** `systemctl list-units --type=service --state=running --no-pager --plain` (filter to non-system services of interest)
7. **Listening ports:** `ss -tlnp` (TCP listeners with process names)
8. **Databases:** Check for running PostgreSQL (`pg_lsclusters` or `systemctl status postgresql`), MySQL (`systemctl status mysql`), Redis (`systemctl status redis`), SQLite files in common locations
9. **Cron jobs:** `crontab -l` for user brucewayne9, plus `ls /etc/cron.d/` and `ls /etc/cron.daily/`
10. **Package manager:** `dpkg --list 2>/dev/null | wc -l` or `rpm -qa 2>/dev/null | wc -l` for package count
11. **Python version:** `python3 --version`
12. **Node version:** `node --version 2>/dev/null`

**Cross-server connections to detect:**
- Check `/etc/hosts` for references to other servers in the 75.43.156.* range
- Check env files in common locations (`~/.env`, `/opt/*/.env`, `/etc/environment`) for references to 75.43.156.* IPs
- Check Docker container environment variables for cross-server references
- Check Traefik config on 117 for backend server references
- Check the health monitor on 105 (local — `scripts/alfred_claw_monitor.py`) for references to other servers

**Script design:**
- Use `subprocess.run(["ssh", alias, command])` for remote execution
- Capture stdout, handle timeouts (30s per command), handle failures gracefully
- Build a dict per server with all collected data
- Include `audit_timestamp` in ISO format
- Include `105` (local machine) in the inventory too — collect same data locally without SSH
- Write output to `data/infrastructure/inventory.json` with `json.dump(indent=2)`

**SSH aliases to use:** `server-98`, `server-100`, `claw`, `server-104`, `lonewolf`, `server-121`, plus `localhost` for 105.
