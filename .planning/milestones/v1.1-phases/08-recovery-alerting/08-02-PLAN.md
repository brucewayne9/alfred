3. **Restore steps** (numbered, specific):

   c. Config file restoration (cp from extracted tarball to original paths)
   d. Database restoration (server-specific: pg_restore for PostgreSQL, mysql < dump for MySQL, cp for SQLite)
   e. Docker container restoration (docker-compose up or docker run from backed-up compose files + volume imports)
   f. Service restart verification
   g. Crontab restoration
