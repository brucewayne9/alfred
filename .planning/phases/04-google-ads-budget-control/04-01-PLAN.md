```python
# Audit log path
AUDIT_LOG_PATH = Path("/home/aialfred/alfred/data/ads_audit.jsonl")

def _audit_log(platform: str, operation: str, entity_id: str, entity_name: str,
               old_value: dict, new_value: dict, customer_id: str = None) -> None:
    """Append one line to the ads audit log (append-only JSONL)."""
    entry = {
        "timestamp": datetime.now(timezone.utc).isoformat(),
        "platform": platform,
        "operation": operation,
        "entity_id": entity_id,
        "entity_name": entity_name,
        "customer_id": customer_id,
        "old_value": old_value,
        "new_value": new_value,
        "requester": "alfred",
    }
    try:
        with open(AUDIT_LOG_PATH, "a") as f:
            f.write(json.dumps(entry) + "\n")
    except Exception as e:
        logger.warning(f"Failed to write audit log: {e}")
```

Note: The audit log must never cause a write operation to fail -- wrap in try/except and log warning only.

**A. `update_campaign_budget(campaign_id, new_daily_budget, customer_id=None)` -- add after `set_campaign_status()`:**

Two-step mutation per RESEARCH.md Pattern 1:
1. Query campaign to get `campaign_budget.resource_name`, `campaign_budget.amount_micros`, `campaign_budget.explicitly_shared`, `campaign_budget.reference_count`, `campaign.name`.
2. If not found, return `{"error": "Campaign {campaign_id} not found"}`.
3. If `explicitly_shared` is True AND `reference_count > 1`, query all campaigns sharing that budget (`SELECT campaign.id, campaign.name FROM campaign WHERE campaign_budget.resource_name = '{budget_resource_name}' AND campaign.status != 'REMOVED'`) and include `shared_campaigns` list in the return value so the LLM can warn Mike.
4. Mutate using `CampaignBudgetService.mutate_campaign_budgets()` with `CampaignBudgetOperation`, field mask `["amount_micros"]`, amount = `int(new_daily_budget * 1_000_000)`.
5. Verification read-back: re-query the campaign budget to confirm `amount_micros` changed.
6. Call `_audit_log("google_ads", "update_campaign_budget", campaign_id, campaign_name, {"daily_budget": old_budget}, {"daily_budget": new_daily_budget}, customer_id)`.
7. Return dict with: `success`, `campaign_id`, `campaign_name`, `old_daily_budget`, `new_daily_budget`, `verified_daily_budget` (from read-back), `is_shared`, `shared_campaigns` (list of {id, name} if shared, else empty list).

**B. `set_ad_group_status(ad_group_id, status, customer_id=None)` -- add after `update_campaign_budget()`:**

Per RESEARCH.md Pattern 2:
1. First query current ad group status and name: `SELECT ad_group.id, ad_group.name, ad_group.status, campaign.id, campaign.name FROM ad_group WHERE ad_group.id = {ad_group_id}`.
2. If not found, return error.
3. Mutate using `AdGroupService.mutate_ad_groups()` with `AdGroupOperation`, field mask `["status"]`, status = `client.enums.AdGroupStatusEnum.PAUSED` or `.ENABLED`. Return error for invalid status values.
4. Verification read-back: re-query `ad_group.status` to confirm change.
5. Call `_audit_log("google_ads", "set_ad_group_status", ad_group_id, ad_group_name, {"status": old_status}, {"status": status.upper()}, customer_id)`.
6. Return dict with: `success`, `ad_group_id`, `ad_group_name`, `campaign_id`, `campaign_name`, `old_status`, `new_status`, `verified_status`.
