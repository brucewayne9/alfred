  <files>core/forge/renderers/topic_clip.py</files>
  <action>
Add caption + branding logic to `topic_clip.py`.

1. `_safe_drawtext(text: str) -> str` — sanitise transcript text for ffmpeg drawtext (pitfall 3: contractions like "it's", "can't" crash unescaped drawtext). Replace `'` (straight apostrophe) with `’` (U+2019), replace `\\` with `\\\\`, replace `:` with `\\:`, replace `%` with `\\%`. Then word-wrap to ~38 chars/line using `textwrap.fill(..., width=38)` and the result's newlines become drawtext line breaks. Keep it a pure string function (easy to unit test).

2. `overlay_captions(body_path: Path, caption_text: str, out_path: Path, font_size: int = 52) -> Path` — overlay a bottom-third caption on an already-9:16 video. Font: the project's Hanken Grotesk is NOT installed on 105 (confirmed `fc-list | grep -i hanken` returns nothing at plan time) — use the verified fallback `FONT_PATH = "/usr/share/fonts/truetype/dejavu/DejaVuSans-Bold.ttf"` (confirmed present). drawtext params: `fontfile={FONT_PATH}:fontsize={font_size}:fontcolor=white:bordercolor=black:borderw=3:x=(w-text_w)/2:y=h-text_h-120:text='{safe}'`. Build the filter with the `_safe_drawtext` output. Run `ffmpeg -y -v error -i {body} -vf "drawtext=..." -c:v libx264 -preset veryfast -crf 23 -c:a copy {out}` (audio can be copied here — it's not being cut). If caption_text is empty/whitespace, skip drawtext and just copy body to out_path. Raise RuntimeError with stderr tail on failure.

Note on caption strategy (open question 2, default): for a multi-segment clip use the FIRST SENTENCE of the first segment's text as the single overlay line (split on `. ` take [0], cap 120 chars). Word-level per-segment timed captions are deferred to Phase 13 — document this in a comment.

For topic_clip the `hook` is the concatenated segment audio; pass caption="" to make_branded since captions are already burned in by overlay_captions (make_branded deliberately skips drawtext anyway).

Add the variant + assembly + render logic:

1. `_build_variant_assemblies(segments: list[dict], variant_count: int) -> list[dict]` — return `variant_count` distinct segment orderings (each item: `{"label": str, "segments": [...]}`). Apply enforce_duration() to the base set FIRST, then derive:
   - V0 "original": curated order as-is
   - V1 "reverse": segments reversed
   - V2 "hook-first": highest `score` segment moved to index 0, rest original order
   - V3 "trimmed": each segment's start_s nudged +0.5s and end_s nudged -0.5s (re-run enforce_duration after, floor windows at 1s)
   - V4+ "shuffle-{i}": `random.Random(i).shuffle(copy)` deterministic per index
   Return exactly `variant_count` (1–10) items; if variant_count==1 return only V0. Each variant's segments must be independent copies (no shared mutation).

2. `_synthesise_visual(caption_seed: str, audio_path: Path, out_path: Path, work_dir: Path) -> Path` — for AUDIO-ONLY sources, build a 9:16 video from a generated still + the concatenated audio (research Option B). Lazy import inside the function: `from scripts.rucktalk_common import run_comfyui_cloud` (NEVER inline PIL — project hard rule). Generate one still: `img = run_comfyui_cloud(caption_seed, width=1080, height=1920)`. If run_comfyui_cloud returns None/falsey, fall back to a solid black background via `color=black:s=1080x1920`. Then: `ffmpeg -y -v error -loop 1 -i {img_or_black} -i {audio_path} -c:v libx264 -preset veryfast -crf 23 -c:a aac -b:a 192k -vf "scale=1080:1920:force_original_aspect_ratio=increase,crop=1080:1920,fps=30,format=yuv420p" -shortest {out_path}`. Return out_path.

3. `assemble_variant(source_path, has_video, variant, caption_text, out_path, work_dir) -> Path` — full single-variant pipeline:
   - cut each segment via `_cut_segment` (plan 01) into work_dir
   - concat via `_concat_segments` -> body (video) OR audio-track (audio-only)
   - if audio-only: `_synthesise_visual(caption_text, concatenated_audio, body, work_dir)`
   - `overlay_captions(body, caption_text, captioned, work_dir)`
   - extract the concatenated audio to a hook file (use `audio.clip_audio` or ffmpeg `-vn`) and call `make_branded(captioned, hook_audio, "", out_path)` for logo + final mux
   - return out_path

4. `render(params: dict, out_path: str | Path) -> Path` — top-level entry matching `film_montage.render(params, out_path)` signature. Reads `params["source_id"]`, `params["segments"]`, `params.get("variant_index", 0)`, `params.get("caption","")`. Resolve source file via `from core.forge import ingest; src = ingest.get_source(source_id)["file_path"]`. Detect has_video. Build the assembly for the given variant_index (the handler in plan 03 calls render once per structural variant). Return the branded master path. (The per-variant loop + multiply lives in the plan-03 handler.)
