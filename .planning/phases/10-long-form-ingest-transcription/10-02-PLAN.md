  <action>
In `core/forge/uploads.py`, add a streaming variant that never holds the whole file in RAM. Keep the existing `save_upload(content: bytes, ...)` for back-compat but add:

    async def save_upload_stream(file, filename: str) -> str:
        """Stream a Starlette UploadFile to disk in 8MB chunks; return the new id."""
        ext = Path(filename or "").suffix.lower()[:12]
        uid = uuid.uuid4().hex
        dest_dir = _root() / uid
        dest_dir.mkdir(parents=True, exist_ok=True)
        dest = dest_dir / f"file{ext}"
        with dest.open("wb") as out:
            while True:
                chunk = await file.read(8 * 1024 * 1024)
                if not chunk:
                    break
                out.write(chunk)
        return uid

In `core/api/forge.py`, rewrite the existing `@app.post("/forge/uploads")` handler:
- Remove `content = await file.read()` (the OOM blocker).
- Call `uid = await uploads.save_upload_stream(file, file.filename or "upload.bin")`.
- After saving, resolve the stored path with `uploads.get_upload_path(uid)`, then:
  - `from core.forge import ingest` → `source_id = ingest.create_source("upload", file.filename or "upload.bin", str(path))`
  - `from core.forge import jobs as forge_jobs` → `job_id = forge_jobs.enqueue("ingest_transcribe", {"source_id": source_id})`
- Return `{"upload_id": uid, "source_id": source_id, "job_id": job_id, "filename": file.filename}`.
- Reject genuinely empty uploads (size 0 after streaming) with HTTPException 400 — check the written file size, not an in-memory `content`.

Do not pull any secret from os.environ here (CLAUDE.md rule). FORGE_UPLOAD_DIR handling already lives in uploads._root().
