<objective>
Build the `ingest_transcribe` job handler — the heart of the phase. It downloads a URL source (full-length, no section window) or reads an uploaded file, extracts 16kHz mono audio, transcribes with faster-whisper (`medium`, CUDA, VAD-filtered, word timestamps), writes completed segments to a disk checkpoint AS IT GOES so a service restart resumes instead of restarting (INGEST-03), labels each segment with the A/B speaker heuristic (INGEST-04), and persists the final transcript to the `transcript_segments` table. Register it in the worker and expose read endpoints.

Purpose: Satisfies success criteria 3 (restart-safe) and 4 (speaker-attributed) and makes the async pipeline real. Restart safety is delivered by re-enqueue idempotency: reconcile_orphans() marks an interrupted job 'error', and the operator's retry re-runs the handler, which fast-paths through the checkpoint.
Output: transcribe handler in ingest.py, handler registration, source/transcript read endpoints.
</objective>

  <name>Task 1: Implement the checkpointed ingest_transcribe handler in ingest.py</name>
  <files>core/forge/ingest.py</files>
  <action>
Add `transcribe_handler(params: dict) -> dict` to `core/forge/ingest.py`. It receives `{"source_id": str, "url": str | None}` (job_id is available via forge_jobs if needed for check_cancel). Build it idempotent and checkpoint-driven so a restart resumes.

Flow:
1. `source_id = params["source_id"]`. Load `ckpt = load_checkpoint(source_id)`. If `ckpt["complete"]` is True, fast-path: ensure segments are persisted (call save_segments from the checkpoint) and return early with `{"transcript_id": source_id, "segment_count": len(ckpt["segments"]), "resumed": True}`. This is the restart fast-path.
2. Resolve the media file — localize any non-local source to disk first (the source row may carry a remote/URL reference, not a local path):
   - If `params.get("url")` AND the source has no local file_path yet: download the FULL source with yt-dlp WITHOUT the section window. Use `clips.resolve_source(url)` to get the target, then build a yt-dlp command modeled on `clips.ytdlp_cmd` but strip any `--download-sections`/SECTION_WINDOW argument. Reuse clips' Node-runtime detection and retry flags. Download into the source's upload dir (create one under FORGE_UPLOAD_DIR via uploads if needed), then `update_source(source_id, file_path=<path>)`. Do NOT call clips.fetch_source() (it caps at 90s).
   - Elif the source kind is `cloud` (params has `cloud_path`, and the stored file_path is a REMOTE Nextcloud path): stream the file down from Nextcloud to local disk in chunks (do NOT use nc.download_file, which buffers the whole 11 GB in RAM — use a chunked/streamed WebDAV GET into the source's upload dir), then `update_source(source_id, file_path=<local path>)`. This is the reliable big-file path (operator dropped the doc in Sources/).
   - Else use the existing `get_source(source_id)["file_path"]` (the uploaded file already on local disk).
3. `update_source(source_id, status="extracting")`. Extract audio: `wav = audio.extract_audio(file_path, <dir>/audio.wav)`. Record duration via `audio.duration_seconds`.
4. `update_source(source_id, status="transcribing", duration_s=dur)`. Load faster-whisper once:
       from faster_whisper import WhisperModel
       model = WhisperModel("medium", device="cuda", compute_type="int8_float16")
   Optional safety per research Pitfall 4: if free VRAM on the 3090 is < ~8 GB, fall back to device="cpu", compute_type="int8". Keep it simple — a try/except around the cuda load that retries on cpu is acceptable.
5. Transcribe with `model.transcribe(str(wav), word_timestamps=True, vad_filter=True, beam_size=5, language="en")`. Iterate the segment generator. For each segment append a dict `{"seq": i, "start_s": seg.start, "end_s": seg.end, "text": seg.text.strip(), "words": [{"word": w.word, "start": w.start, "end": w.end} for w in (seg.words or [])]}` to `ckpt["segments"]`. After every N segments (e.g. N=10) call `write_checkpoint(source_id, ckpt)` and `forge_jobs.check_cancel()` at the segment boundary so cancel/restart leaves a fresh checkpoint. The faster-whisper generator only emits not-yet-produced segments, so on a resumed run the already-checkpointed segments are simply re-appended — guard against duplicate seq by seeding the loop index from len of already-done segments OR (simpler and robust) always restart transcription from scratch but skip writing if checkpoint already complete; document whichever you choose. The disk checkpoint is the durability guarantee; perfect mid-file resume of the whisper generator is NOT required — fast-path-on-complete + frequent checkpoint writes satisfy INGEST-03.
6. Run `labelled = assign_speakers(ckpt["segments"])` to add speaker A/B (INGEST-04). Set `ckpt["segments"] = labelled`, `ckpt["complete"] = True`, write_checkpoint.
7. `save_segments(source_id, labelled)` to persist into transcript_segments. `update_source(source_id, status="done", language=info.language if available)`.
8. On any exception: `update_source(source_id, status="error", error=str(e))` and re-raise (so the job row also goes to error).
9. Return `{"transcript_id": source_id, "segment_count": len(labelled), "resumed": False}`.

Honor CLAUDE.md: no secrets from os.environ; the medium model is local-cached (no token needed). Imports added here: `faster_whisper.WhisperModel` (lazy import inside the function to avoid loading CUDA at module import), plus `core.forge.audio`, `core.forge.clips`, `core.forge.jobs as forge_jobs`.
