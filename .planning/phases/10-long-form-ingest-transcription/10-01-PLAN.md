  <files>core/forge/db.py</files>
  <action>
Extend `init_db()`'s `executescript` block (it already creates `jobs` and `trash`) to also create the two tables below. Use `CREATE TABLE IF NOT EXISTS` so the migration is idempotent and re-running on the live DB (data/forge_live.db / FORGE_DB_PATH) is safe.

    CREATE TABLE IF NOT EXISTS sources (
        id           TEXT PRIMARY KEY,
        kind         TEXT NOT NULL,            -- 'upload' | 'url'
        spec         TEXT NOT NULL,            -- original filename or URL
        file_path    TEXT,                     -- absolute path on disk
        status       TEXT NOT NULL DEFAULT 'pending',  -- pending|extracting|transcribing|done|error
        duration_s   REAL,
        language     TEXT,
        error        TEXT,
        created_at   INTEGER NOT NULL,
        updated_at   INTEGER NOT NULL
    );
    CREATE INDEX IF NOT EXISTS idx_sources_status ON sources(status, created_at);

    CREATE TABLE IF NOT EXISTS transcript_segments (
        id           INTEGER PRIMARY KEY AUTOINCREMENT,
        source_id    TEXT NOT NULL REFERENCES sources(id) ON DELETE CASCADE,
        seq          INTEGER NOT NULL,
        start_s      REAL NOT NULL,
        end_s        REAL NOT NULL,
        text         TEXT NOT NULL,
        speaker      TEXT,
        words        TEXT
    );
    CREATE INDEX IF NOT EXISTS idx_segments_source ON transcript_segments(source_id, seq);

Do not change the existing `jobs`/`trash` definitions or the WAL pragma. Keep the same connection-acquisition pattern already used in db.py (do not hand-roll a new sqlite3.connect with different settings — reuse the existing helper/context manager).

<task type="auto">
  <name>Task 2: Create core/forge/ingest.py — source CRUD, checkpoint helpers, speaker heuristic</name>
  <files>core/forge/ingest.py</files>
  <action>
Create `core/forge/ingest.py` with these functions. Reuse the existing Forge DB connection helper from `core/forge/db.py` (same pattern as jobs.py uses) — do NOT open a fresh sqlite3 connection with different pragmas, and do NOT read FORGE_DB_PATH from os.environ directly here; let db.py own the path.

Source CRUD:
- `create_source(kind: str, spec: str, file_path: str | None = None) -> str` — insert a row into `sources` with a fresh `uuid.uuid4().hex` id, status 'pending', created_at/updated_at = int(time.time()); return the id.
- `get_source(source_id: str) -> dict | None` — fetch one source row as a dict.
- `update_source(source_id: str, **fields) -> None` — UPDATE arbitrary columns (status, duration_s, language, error, file_path) plus updated_at. Whitelist the column names to avoid SQL injection.
- `save_segments(source_id: str, segments: list[dict]) -> None` — DELETE existing transcript_segments for this source_id then bulk INSERT the provided segments (seq, start_s, end_s, text, speaker, words). `words` is a JSON-serialized list; json.dumps it if it is not already a string. Make this idempotent (safe to call repeatedly with the full segment list).

On-disk checkpoint helpers (these are what make transcription restart-safe — INGEST-03):
- `_checkpoint_dir() -> Path` — returns `Path("data/forge_transcripts")` (mkdir parents=True, exist_ok=True). Respect a `FORGE_TRANSCRIPT_DIR` env override the same way uploads.py respects FORGE_UPLOAD_DIR.
- `checkpoint_path(source_id: str) -> Path` — `_checkpoint_dir() / f"{source_id}.json"`.
- `load_checkpoint(source_id: str) -> dict` — read+json.loads the checkpoint file; if missing, return `{"segments": [], "complete": False}`.
- `write_checkpoint(source_id: str, data: dict) -> None` — json.dump to a temp file then os.replace onto the final path (atomic, so an interrupted write never corrupts the checkpoint).

Speaker heuristic (INGEST-04) — copy the proven approach from scripts/notebooklm_clip.py:165-187:
- `SHORT_SEGMENT_THRESHOLD_S = 1.2`
- `assign_speakers(segments: list[dict]) -> list[dict]` — alternate "A"/"B" per segment, inherit the prior label when a segment is shorter than the threshold. Return new dicts with a `speaker` key added (do not mutate inputs).

Imports: `json`, `os`, `time`, `uuid`, `tempfile`, `pathlib.Path`. Do not import faster-whisper or ffmpeg here — this module is pure storage/util so Plan 10-03 can extend it with the transcription handler.
  </action>
