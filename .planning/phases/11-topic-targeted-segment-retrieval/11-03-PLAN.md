<objective>
Add a "Topic" tab to the Forge UI where the operator picks a transcribed source, types a topic in plain language, and sees a ranked list of matching segments — each with in/out timestamps, a relevance score, and the transcript text inline. The operator can deselect any match and adjust its in/out trim. The curated list is the hand-off to Phase 12 (held client-side; this phase does NOT cut clips).

Purpose: Completes TOPIC-01 (operator sees ranked timestamped results in the UI), TOPIC-02 (reads transcript text inline to verify on-topic), TOPIC-03 (deselect/trim before assembly). This plan OWNS all edits to services/forge-web/index.html for this phase.
Output: a new Topic tab in services/forge-web/index.html, plus a human-verified precision check on a REAL transcript.
</objective>

<task type="auto">
  <name>Task 2: Wire search, ranked previewable results, and curate (deselect/trim)</name>
  <files>services/forge-web/index.html</files>
  <action>
In the page `<script>`, add the Topic-tab logic. Hold selection state in a module-level object `topicState = { results: [], selected: {} }` keyed by `win_id` (or `seq_start` if win_id isn't returned — the API returns seq_start/seq_end; build a stable key `${start_s}-${end_s}`).

Functions:
- `loadTopicSources()`: `fetch(API+'/forge/sources?status=done',{credentials:'include'})` -> populate `#topicSource`. Call on first switch to the Topic tab (hook into tabTo or lazy-load when the tab opens) and on the Refresh button.
- `fmtTime(s)`: seconds -> `M:SS` for display (e.g. 142.5 -> "2:22").
- `runTopicSearch()`: read source id, query, speaker (omit if "Any"), top_k, threshold. Guard empty query/source with toast(). Build URL: `API+'/forge/sources/'+encodeURIComponent(id)+'/search?q='+encodeURIComponent(q)+'&top_k='+k+'&threshold='+t+(speaker?'&speaker='+speaker:'')`. fetch with credentials. On 409 toast "Source still processing"; on 404 toast "Source not found". Render `data.results` via renderTopicResults().
- `renderTopicResults(results)`: if empty, show `<div class="empty">No segments matched — try a broader topic or lower the threshold.</div>`. Otherwise for each result render a card with:
  * A checkbox (checked by default) -> toggles `topicState.selected[key]` (TOPIC-03 deselect).
  * Header: `▶ {fmtTime(start_s)} – {fmtTime(end_s)}` and a relevance badge showing `Math.round(score*100)+'%'` (NEVER raw distance) and speaker if non-empty.
  * The transcript `text` shown inline (TOPIC-02) — wrap long text in a scrollable/clamped block using existing styles; this is the on-topic verification surface, so it must be readable without a second click (a collapsible `<details>` is acceptable but default-open or a clamp with "show more").
  * Two number inputs "In" / "Out" pre-filled with start_s/end_s (seconds, step 0.1) -> on change, write the trim into `topicState.selected[key].trimIn/trimOut`, clamped within [start_s, end_s] (TOPIC-03 trim). Default trim = full window.
  * Initialize `topicState.selected[key] = { ...result, kept:true, trimIn:start_s, trimOut:end_s }` for every rendered result.
- After render and on every toggle/trim change, call `updateTopicSelectionSummary()`: count kept items, write `#topicSelected` text like `4 of 10 segments selected`.
- "Copy selection" button: build the kept list `[{start_s:trimIn, end_s:trimOut, text, speaker, score}]` and `JSON.stringify` it to the clipboard (navigator.clipboard) + toast "Selection copied — ready for assembly". This is the explicit Phase-12 hand-off (no clip cutting here).

Escape all text with the existing esc(). Match existing card classes. Keep it dependency-free vanilla JS like the rest of the file.
