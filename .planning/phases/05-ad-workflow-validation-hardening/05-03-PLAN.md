  <action>
Create `scripts/validate_meta_ads.py` — a standalone script that imports and calls each of the 22 registered Meta Ads tool functions from `integrations/meta_ads/client.py` against live campaigns.

**Structure:**
```python
#!/usr/bin/env python3
"""Validate all 22 Meta Ads tools against live campaigns."""
import sys
import json
import time
from pathlib import Path

# Ensure project root is in path
sys.path.insert(0, str(Path(__file__).parent.parent))

from integrations.meta_ads.client import (
    is_connected,
    get_ad_account_info,
    list_campaigns,
    # ... all functions
)
```

**Validation flow (per user decision — reads first, then writes):**

1. **Connectivity check:** Call `is_connected()`. If False, abort with clear error.

2. **Discover campaign IDs dynamically:** Call `list_campaigns()` to find Rod Wave and One Music Festival campaign IDs. Store them for use in subsequent tool calls. If no ACTIVE campaigns found, use PAUSED ones and note it.

3. **Read tools (validate all ~14 read tools):**
   For each read tool, call it with appropriate parameters derived from discovered campaign data. Log pass/fail. On failure: log the error, continue to next tool (per user decision — don't halt).

   Read tools to validate (all 14):
   - `get_ad_account_info()`
   - `get_account_summary()` (or equivalent account-level function)
   - `list_campaigns()`
   - `get_campaign_details(campaign_id)`
   - `get_campaign_insights(campaign_id)` or similar performance function
   - `list_ad_sets(campaign_id)`
   - `get_ad_set_details(adset_id)`
   - `list_ads(adset_id)`
   - `get_ad_details(ad_id)`
   - `get_ad_creative(ad_id)` or creative function
   - `get_performance_summary()` or account-level insights
   - Any remaining read functions up to the full list

   NOTE: The exact function names must be read from client.py at execution time. The research says 22 tools total (14 read + 8 write). Map each registered tool name in `core/tools/registry.py` TOOL_CATEGORIES["meta_ads"] to its client.py function.

4. **Write tools — status changes (validate 6 status tools):**
   Use the change-and-revert pattern:
   - Find one ACTIVE ad set or campaign (from list_campaigns/list_ad_sets results)
   - Pause it → verify PAUSED status in response (verified_status field from Plan 02)
   - Re-enable it → verify ACTIVE status in response
   - This validates pause_campaign, enable_campaign, pause_ad_set, enable_ad_set (and pause_ad, enable_ad if an active ad exists)
   - If no suitable ACTIVE entity exists for a given level (ad/adset/campaign), log as SKIPPED with reason

5. **Write tools — budget changes (validate 2 budget tools):**
   Use the change-and-revert pattern per user decision:
   - Read current budget via campaign/adset details
   - Increase by $1: `update_campaign_budget(campaign_id, current + 1)`
   - Verify the response contains `verified_budget` close to current + 1
   - Revert: `update_campaign_budget(campaign_id, current)` — restore original
   - Verify the response contains `verified_budget` close to original
   - Repeat for `update_ad_set_budget` with an ad set

6. **Report:** Print a summary table to stdout:
   ```
   Meta Ads Tool Validation Report
   ================================
   Date: 2026-02-21
   API Version: v22.0
   Account: act_XXXXX

   Tool                          | Status | Notes
   ------------------------------|--------|------
   get_ad_account_info           | PASS   |
   list_campaigns                | PASS   | Found N campaigns
   pause_campaign                | PASS   | Verified PAUSED
   ...

   Results: XX/22 PASS, X FAIL, X SKIPPED
   ```

   Also write the report as JSON to `data/meta_ads_validation.json` for record-keeping.

**Error handling:** Each tool call wrapped in try/except. On exception: log FAIL with error message, continue. At the end, report all failures. Per user decision: don't halt on individual tool failure.

**Important:** Read the actual function signatures from client.py before calling them. The research lists approximate names — use the real names as they exist after Plan 01 and 02 modifications.
