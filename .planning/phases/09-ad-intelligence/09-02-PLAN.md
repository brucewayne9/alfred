Create `integrations/ad_intelligence/suggestions.py` with a `generate_suggestions(days: int = 7) -> dict` function that:

1. Calls `get_cross_platform_summary(days)` from the cross_platform module (Plan 01 dependency)
2. Analyzes the combined data using rule-based heuristics (NOT calling an external LLM — this runs as a tool the LLM already has context for)
3. Returns a dict with:
   ```python
   {
       "period_days": days,
       "suggestions": [
           {
               "priority": "high" | "medium" | "low",
               "type": "pause" | "budget_increase" | "budget_decrease" | "reactivate" | "investigate",
               "platform": "meta" | "google" | "cross_platform",
               "campaign_name": str,
               "campaign_id": str,
               "reason": str,  # Plain English explanation
               "metric_detail": str,  # The specific numbers backing the suggestion
               "suggested_action": str,  # What to do, e.g. "Pause this campaign" or "Increase daily budget from $20 to $35"
           },
           ...
       ],
       "summary": str,  # 1-2 sentence overview for Alfred to relay
   }
   ```

**Heuristic rules to implement** (each generates a suggestion if triggered):

a. **High CPC outlier** (priority: high, type: investigate): If a campaign's CPC > 2x the platform average CPC, flag it. Reason: "CPC of ${cpc} is {x}x the platform average of ${avg}." Suggested action: "Review targeting and ad creative for this campaign."

b. **Zero conversions with significant spend** (priority: high, type: pause): If a campaign spent > $10 in the period but has 0 conversions, flag it. Reason: "Spent ${spend} with zero conversions over {days} days." Suggested action: "Consider pausing this campaign or revising the conversion setup."

c. **Low CTR** (priority: medium, type: investigate): If campaign CTR < 0.5% (for Meta) or < 1.0% (for Google Search), flag it. Reason: "CTR of {ctr}% is below the {threshold}% threshold." Suggested action: "Test new ad copy or adjust audience targeting."

d. **Strong performer underfunded** (priority: medium, type: budget_increase): If a campaign has CPC below platform average AND CTR above platform average AND conversions > 0, flag it. Reason: "This campaign is outperforming on CPC (${cpc} vs ${avg}) and CTR ({ctr}% vs {avg}%)." Suggested action: "Consider increasing the daily budget to capture more of this high-performing traffic."

e. **Paused campaign with good historical metrics** (priority: low, type: reactivate): If a campaign is PAUSED but had CTR > platform avg and CPC < platform avg in its last active period, flag it. Reason: "This paused campaign had strong metrics when active." Suggested action: "Consider reactivating this campaign."

f. **Budget imbalance across platforms** (priority: medium, type: cross_platform): If one platform accounts for > 80% of total spend but < 50% of total conversions, flag it. Reason: "Meta accounts for {x}% of spend but only {y}% of conversions." Suggested action: "Consider shifting budget from {over_platform} to {under_platform}."

g. **Declining performance** (priority: high, type: investigate): Compare 7-day vs 14-day metrics (if days >= 14). If CPC increased by > 30% or CTR decreased by > 30% in the recent 7 days vs prior 7 days, flag it. Only implement this if `days >= 14`; skip if not enough data. If days < 14, simply skip this heuristic without error.

For each heuristic:
- Use safe division (avoid ZeroDivisionError) — if denominator is 0, skip that check
- Only generate suggestions for campaigns that have data (skip campaigns with 0 impressions unless flagging spend-with-no-conversions)
- Sort suggestions by priority: high first, then medium, then low

Generate the `summary` field:
- Count high/medium/low suggestions
- Example: "Found 2 high-priority and 3 medium-priority optimization opportunities across Meta and Google campaigns."
- If no suggestions: "All campaigns are performing within normal parameters. No immediate action needed."

Use `logging.getLogger(__name__)` for debug logging of which heuristics triggered.
