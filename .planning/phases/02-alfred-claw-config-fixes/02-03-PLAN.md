5. Edit `~/.openclaw/openclaw.json` — update the `agents.defaults.memorySearch` section (currently empty `{}`) to:
   ```json
   {
     "provider": "openai",
     "model": "nomic-embed-text",
     "remote": {
       "baseUrl": "http://127.0.0.1:11434/v1/",
       "apiKey": "ollama-local"
     },
     "fallback": "none"
   }
   ```
   Use `provider: "openai"` because Ollama exposes an OpenAI-compatible `/v1/embeddings` endpoint.
   Use `fallback: "none"` because the OpenAI key is dead (HTTP 401).

6. Validate JSON: `python3 -c "import json; json.load(open('/home/brucewayne9/.openclaw/openclaw.json'))" && echo "JSON valid"`

**INFRA-04 — Log cleanup:**

7. Create a cron cleanup script (simpler than logrotate for date-based /tmp/ log files):
   ```bash
   sudo tee /etc/cron.daily/openclaw-log-cleanup << 'CRON'
   #!/bin/bash
   # Clean up openclaw logs older than 7 days
   find /tmp/openclaw-1000/ -name "*.log" -mtime +7 -delete 2>/dev/null
   find /tmp/openclaw/ -name "*.log" -mtime +7 -delete 2>/dev/null
   CRON
   sudo chmod +x /etc/cron.daily/openclaw-log-cleanup
   ```
   A cron.daily script is preferred over logrotate because OpenClaw already creates new files per day — the issue is just cleanup of old files. logrotate handles /tmp/ glob paths poorly.

12. Restart the openclaw supervisor:
    ```bash

    pkill -f "^openclaw$" || pkill -f "openclaw gateway"
    ```
